11. Operators

An operator is something that acts on other values take the following example.

x = 4 + 2

The numbers 4 and 2 are the operands and the + is the operator. Python supports the following types of operators, which are similar to those in most other programming languages.

Arithmetic Operators

These carry out mathematical functions.

Addition, adds 2 values together

a = 3 + 4 # a will equal 7

Subtraction, subtracts one number from another

b = 4 – 5 # b will equal -1

Multiplication, multiplies one number by another

c = 7 * 2 # c will equal 14

Division, divides one number by another

d = 8 / 2 # d will equal 4

Exponent, raises one number to the power of another.

E = 2 ** 10 # e will equal 1024

Modulus, returns the remainder when one value is divided by another

f = 11 % 5 # f will equal 1

Floor Division, the fractional value is removed after the division

g = 11.0 // 5.0 # g will equal 2.0

Comparison Operators

Comparison operators allow your programs to make decisions.

Equal evaluates to True if the operands are equal.

a = 2 == 2 # a will equal True

Not equal evaluates to True if the operands are not equal

b = 2 != 2 # b a will equal False

Less than evaluates to True if the value on the right is larger than the value on the left.

c = 2 < 3 # c will equal True

Greater than evaluates to True if the value on the left is larger than the value on the right.

d = 4.6 > 4.1 # d will equal True

Less than or equal evaluates to True if the value on the right is larger than or equal to the value on the left.

e = 4.1 <= 4.1 # e will equal True

Larger than or equal evaluates to True if the value on the left is larger than or equal to the value on the right.

f = 4.6 >= 4.1 # f will equal True

Assignment Operators

These operators assign values to a variable.

Simple assignment the operand on the right to the variable on the left.

a = 9

Add and assigns the right operand to the variable on the left and assigns the value to the variable.

b += 3

Subtracts and assigns the right operand from the variable on the left and assigns the value to the variable.

c -= 3

Multiplies and assigns the right operand by the variable on the left and assigns the value to the variable.

d -= 3

Divides and assigns the variable on the left by the right operand and assigns the value to the variable.

e /= 3

Modulus and assign takes the modulus of the variable on the left with the operand on the right and assigns the value to the variable.

f %= 10

Exponential and assign performs exponential calculation modulus of the variable on the left with the operand on the right and assigns the value to the variable.

g **= 10

Floor and assign performs a floor division of the variable on the left by the right operand and assigns the value to the variable.

h //= 3

Bitwise Operators

Bitwise operators operate on bits within a variable on a bit by bit basis.

AND operator if the corresponding bits in both operands are set then the result for that bit will be that it is set.

a = 6 & 2 # a will equal 2

OR operator if the corresponding bit in either operand is set then the result for that bit will be that it is set

b = 6 | 3 # b will equal 7

XOR operator if the corresponding bit is set in one but not both of the operands then the result for that bit will be that it is set.

c = 6 ^ 3 # c will equal 5

Ones Complement Operator inverts all the bits in the operand.

d = ~170 # d will equal 85

Left shift operator, shifts all the bits in the left hand operand to the left by the value of the right hand operator.

e =6 << 3 # e will equal 48

Right shift operator, shifts all the bits in the left hand operand to the right by the value of the right hand operator.

f = 170 >> 1 # f will equal 85

Logical Operators

These are usually used to combine more than one logical operation.

Logical AND operator if both the operands are true then it evaluates to true.

(value1 < value2) and (value3 > value4)

Logical OR operator if either of the operands are true then evaluates to true.

(value1 == value2) and (value3 > value4)

Logical NOT operator inverts the value, if the value is true then it will evaluate to false.

not(value1) = false

Membership Operators

Membership operators test for membership in a sequence such as list, tuples or strings.

To find if a variable is in the specified sequence the in operator is used to find out if a variable is not in a specified sequence the not in operator is used

The following example shows the membership operators in action.

# File: membershipOperatorExample.py

 

myTuple = ("Jon", 22, "Dave", 33, "Sarah", 33, "Mark", 45)

 

if "Jon" in myTuple:

print("Jon in tuple")

 

if 66 not in myTuple:

print("66 not in tuple")

 

if 33 in myTuple:

print("33 in tuple")

Identity Operators

Identity operators compare the location were two objects are held in memory.

The is operator evaluates to true if the variables are bound to the same object and the is not operator can be used to check two variables are not bound to the same object.

The following example creates two lists and checks that they are separate objects it then makes them the same object.

# File: identityOperatorsExample.py

 

myList1 = [1, 2, 3]

myList2 = [1, 2, 3]

 

# The lists hold the same values but are different objects

if myList1 is myList2:

print("Lists are the same object 1")

 

# Make list1 point to myList2

myList1 = myList2

 

# Check the lists are the same object

if myList1 is myList2:

print("Lists are the same object 2")

 

# Prove the lists point to the same object

myList2[2] = 999

print(myList1[2])

Operator Precedence

The operators have the following order of precedence from high to low.

Exponential

**

Complement, unary plus and minus

~, +, -

Multiplication, division, modulus, floor division

*, /, %, //

Addition and subtraction

+, -

Bitwise left and right shift

<<, >>

Bitwise

AND, OR, XOR, ^, |, &

Comparison operators

<=, <, >, >=

Equality operators

!=, ==

Assignment operators

=, +=, -=, *=, /=, %=, //=, **=

Identity operators

is not, is

Membership operators

in not, in

Logical Operators

and, or, not

It might not seem immediately obvious why this is important but consider the following

x = 4 + 5 * 10

according to the order of precedence this will equate to 54 where as it might have meant it to equal 90. In general unless its immediately obvious what you mean then always use brackets.

x = (4 + 5) * 10

x = 4 + (5 * 10)

Using brackets means that the formula will always get interpreted in the way that you want it to be. To avoid confusion also use brackets for logic expressions the following example should demonstrate why this is important

if a < 10 and a > 5 or a == 5:

In the above expression it is unclear what the actual intention is, using brackets will clarify the exact intention of the expression.

if ((a < 10) and (a > 5)) or (a == 5):

if (a < 10) and ((a > 5) or (a == 5)):

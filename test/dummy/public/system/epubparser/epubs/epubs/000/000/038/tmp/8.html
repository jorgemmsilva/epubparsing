7. Python Syntax

The syntax defines how scripts should be written so that the interpreter can understand the code you are writing. Incorrect syntax will cause your script to produce errors and will slow the development time so take some time to understand the syntax before starting to code. The coding standard that most people write to is known as PEP8 this isn't as onerous as it sounds and all the examples in the book are written to conform to the standard. There are various online tools that allow you to check you code some IDEs such as PyCharm will inspect and report on code that does not follow the PEP8 specification.

Identifiers in Python

A Python identifier is a name used to define a variable, function or other object. In Python identifiers must start with an upper / lower case letter or an underscore (_), the rest of the identifier can contain upper / lower case letters, under scores or numeric characters. Python is a case sensitive language so the identifiers ResultValue and resultvalue will be interpreted as being different.

There are some conventions which are followed by Python programmers.

Class names start with upper case letters all other identifiers should start with a lower case letter.

Private identifiers start with a leading underscore.

Strongly private identifiers start with two leading underscores.

Language defined special names end in two trailing underscores.

Reserved words (see Appendix 2) may not be used as identifier names.

Program Layout

One of the main differences between Python and other programming languages is there are no braces to indicate functions, blocks of code or class definitions. Line indentation is used to define these blocks instead. The indentation can use spaces or tabs, the number of spaces or tabs that are used for the indentation can be variable but all statements in the block must be indented by the same amount, in the following code the line print (“Value still False”) will create an error as it is indented by an extra space.

if True:

print ('Value is True')

print ('Value still True')

else:

print ('Value is False')

print ('Value still False')

In Python a line of code ends with a new line, if you want to split a line of code across several lines then the a back slash (\) can be used as a line continuation character.

Value = Var1 + \

1000 * \

Var3

Definitions which are enclosed with [], {} or () can be split across lines without the continuation character.

Staff = [“John”, “Simon”, “Tan”

“Alan”, “Dave”]

String literals can be enclosed in single('), double(“) or triple quotes ('”, “”), the triple quotes allow the string to span multiple lines.

String1 = 'Test'

String2 = “Test String”

String3 = “'Split string across

multiple lines”'

Multiple statements can be entered onto the same line as long as there is no requirement for a new code block.

X = 20: print(X)

Blocks of code made up from individual statements are called suites. If a statement is compound or complex (if, while, def, class) then it is terminated by a colon followed by a suite. The following shows the basic syntax of the while statement.

while expression:

suite

Comments

Comments are indicated with a # outside of a string literal, anything after a #will be ignored until a new line is detected.

# This is a comment

print ('Comment Test') #This is a comment after a statement
